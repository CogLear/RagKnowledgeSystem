## 三、导入操作流程

### 3.1 方式一：通过前端界面上传（推荐）

1. 登录系统，进入 **知识库管理** 页面
2. 选择目标知识库：
   - **集团信息化**（kbId: 1997855927072321537）
   - **业务系统**（kbId: 1997857139737882625）
3. 点击「上传文档」按钮
4. 选择对应文件夹下的 `.md` 文件
5. 设置分块策略（推荐：按段落自然分块，chunk size 500）
6. 确认上传，等待向量化处理完成
7. 上传完成后，进入对话测试验证召回效果

**对应关系提醒**：
- `集团信息化/人事/*.md` → 上传到「集团信息化」知识库
- `集团信息化/IT支持/*.md` → 上传到「集团信息化」知识库
- `集团信息化/财务/*.md` → 上传到「集团信息化」知识库
- `业务系统/OA系统/*.md` → 上传到「业务系统」知识库
- `业务系统/保险系统/*.md` → 上传到「业务系统」知识库

### 3.2 方式二：通过 API 批量导入

```bash
# 示例：使用 curl 上传单个文档
curl -X POST "http://localhost:9090/api/ragent/knowledge/document/upload" \
  -H "Authorization: Bearer ${TOKEN}" \
  -F "file=@人事制度.md" \
  -F "kbId=1997855927072321537" \
  -F "chunkSize=500"
```

### 3.3 方式三：通过后台管理导入

1. 启动 Spring Boot 应用
2. 访问管理后台（通常为 `/admin` 路径）
3. 进入「知识库管理」→「文档导入」
4. 选择知识库，点击「批量导入」
5. 选择整个文件夹，系统将自动按文件夹名称匹配知识库

## 四、系统交互节点配置（SYSTEM 类型）

### 4.1 什么是 SYSTEM 类型节点

SYSTEM 类型是意图树中的一种特殊节点类型，当用户问题匹配到 SYSTEM 类型的意图时：
- **不会**进行知识库检索（RAG）
- **直接**使用节点配置的 `promptTemplate` 回复用户
- 适用于：欢迎语、关于助手、通用寒暄等场景

### 4.2 预定义的 SYSTEM 类型节点

| 节点 ID | 名称 | 匹配问题示例 | 建议配置 |
|--------|------|-------------|---------|
| sys-welcome | 欢迎与问候 | 你好、早上好、在吗、嗨 | 友好的欢迎语 |
| sys-about-bot | 关于助手 | 你是什么、你是谁、你能做什么 | 助手自我介绍 |

### 4.3 配置方法

1. 启动前端和后端服务
2. 访问 `/admin/intent-tree` 进入意图树管理页面
3. 找到对应的 SYSTEM 类型节点（可通过筛选类型找到）
4. 点击编辑按钮，填写 `promptTemplate` 字段

### 4.4 promptTemplate 配置示例

**sys-welcome 欢迎与问候：**
```markdown
你好！👋 我是智能助手，可以帮助你：

• 解答问题 — 咨询公司制度、业务流程等
• 知识检索 — 查找 OA 系统、保险系统等文档
• 业务处理 — 通过 MCP 工具完成特定任务

有什么可以帮你的吗？
```

**sys-about-bot 关于助手：**
```markdown
我是基于 RAG 技术的智能问答助手，主要功能包括：

• **知识问答** — 基于企业知识库回答各类问题
• **意图识别** — 理解用户问题并路由到对应知识库
• **深度思考** — 开启深度思考模式可获得更深入的分析

输入你的问题，我来帮你解答！
```

### 4.5 配置注意事项

- `promptTemplate` 支持 Markdown 格式
- 可以使用 `\n` 表示换行
- SYSTEM 节点优先于 KB 节点匹配（同类问题不要重复配置）
- 配置后无需重启服务，刷新页面即可生效

## 五、导入注意事项

### 5.1 文件格式

- **推荐格式**：`.md`（Markdown）
- 其他支持格式：`.txt`、`.pdf`、`.docx`、`.html`
- 注意：文档内容需为 UTF-8 编码，否则可能出现乱码

### 5.2 文件名规范

- 文件名建议使用中文，便于管理和识别
- 避免特殊字符：`/ \ : * ? " < > |`
- 建议格式：`文档标题.md`

### 5.3 知识库对应

- 每个文档必须上传到对应的知识库
- **集团信息化**知识库（kbId: `1997855927072321537`）：
  - 包含：人事、IT支持、财务相关的文档
- **业务系统**知识库（kbId: `1997857139737882625`）：
  - 包含：OA系统、保险系统相关的文档

### 5.4 向量化参数建议

| 参数 | 推荐值 | 说明 |
| --- | --- | --- |
| chunkSize | 500 | 每个分块的目标字符数 |
| chunkOverlap | 50 | 分块重叠字符数，保证上下文连续 |
| embeddingModel | 优先使用 BGE 或 M3E | 向量化模型，影响召回效果 |

### 5.5 常见问题

| 问题 | 解决方案 |
| --- | --- |
| 文档上传后无法召回 | 检查向量化任务是否完成；确认 chunkSize 是否合理 |
| 召回内容不准确 | 检查文档内容是否与意图节点描述匹配；尝试调整 chunkOverlap |
| 文档上传失败 | 检查文件格式和大小；确认知识库 ID 是否正确 |
| 分块后语义不完整 | 减小 chunkSize；检查 Markdown 标题层级是否清晰 |

## 七、验证测试建议

上传完成后，建议进行以下验证测试：

1. **单点召回测试**：用示例问题验证是否能召回正确文档
2. **跨知识库测试**：验证不同知识库的文档不会相互干扰
3. **边界测试**：用同义词、口语化表达测试召回鲁棒性
4. **上下文测试**：验证多轮对话中，历史信息是否影响召回

**示例测试问题**：

| 测试问题 | 期望召回文档 |
| --- | --- |
| 请假流程是怎样的？ | 人事制度.md |
| 试用期多久转正？ | 人事制度.md |
| 公司VPN连不上怎么办？ | IT支持.md |
| 发票抬头有哪些？ | 开票信息.md |
| 差旅报销需要哪些资料？ | 差旅报销指南.md |

## 八、后续维护

- 新增文档时，按上述文件夹结构放置到对应目录

- 删除文档时，同步清理向量数据库中的对应数据
- 定期检查召回效果，根据实际使用情况调整分块策略
